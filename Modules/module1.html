Layout and Grid

Overview

One of Bootstrap's key strength is its powerful grid system that allows you to make your website responsive easily. To delve inside this, we'll discuss breakpoints, the grid system, and layout utilities included in Bootstrap.

In the further modules, we'll be relying on you to use your browser in developer tools to further see how the html and css code is implemented.
Developer tools (inspect element) is very useful when creating web pages as it allows you to edit and view html/ css code in your browser. It can also help you develop for different screen sizes and network speeds. Most importantly, it looks really cool.

Breakpoints

Bootstrap uses a mobile first approach in its development. To achieve this, Bootstrap follows the following media queries that you match with Bootstrap's layouting capabilities

  • xs
  • sm
  • md
  • lg
  • xl

If you're not familiar with media queries, don't worry as we'll use these and show examples below.

Containers

Now for the interesting part, we'll utilize Bootstrap's grid system. However, before we start we'll have to use containers.

The container class is added to elements such as sections and div for you to use the Bootstrap grid system. There are three types of containers as per example below. To see the difference, please adjust your window size or open your browser's developer tools.

All in one

The default .container class is a responsive, fixed-width container, meaning its max-width changes at each breakpoint

Fluid

Use .container-fluid for a full width container, spanning the entire width of the viewport.

Responsive

.container-xs/sm/md/lg are used if you only want to have the container at full width until a specific breakpoint

100% wide until small breakpoint
100% wide until medium breakpoint
100% wide until large breakpoint
100% wide until extra large breakpoint
